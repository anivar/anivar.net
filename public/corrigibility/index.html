Corrigibility — the structural precondition for public infrastructure that can be corrected by the people it governs.

Two papers. One invariant. Five jointly necessary tests — EXIT, CODE, AUDIT, GOVERN, FORK — that close a corrective feedback loop. Removing any one opens the loop, and correction becomes discretionary.

Thesis

Digital Public Infrastructure is not merely a collection of software stacks; it is the encoding of political arrangements into technical artifacts. Because these systems apply fixed rules to the infinite variety of human life, they will inevitably misclassify, exclude, or fail specific users. The critical question is not whether errors occur, but whether the architecture permits those affected to detect, correct, and reverse them before harm becomes permanent.
— Paper I, Introduction

The framework is governance-agnostic but politically pointed. Systems promoted globally as model DPI — Aadhaar, UPI, large parts of the India Stack export agenda — fail one or more of the five tests when measured structurally. Systems that already run the world — Linux, Kubernetes, the Web — pass them. The framework is the apparatus that makes that comparison falsifiable rather than rhetorical.

The Invariant

Corrigibility does not guarantee fairness. It guarantees reversibility — the architectural capacity for affected participants to detect error, signal harm, and trigger correction. Five conditions form a closed corrective loop; the failure of any one converts the system to open-loop, in which errors accumulate without bound. Partial compliance is functionally equivalent to complete failure.

Figure 1. Four control layers form a closed feedback loop. Each layer plays a distinct cybernetic role: CODE/AUDIT as sensor, EXIT as error signal, GOVERN as actuator, FORK as selection pressure over the loop itself. Removing any layer opens the loop — the framework's fatal failure property. Derived from Ashby's Law of Requisite Variety (1956): a controller must possess at least as many response states as the system it regulates.

The Five Tests

EXIT Reversibility of Participation DPI ✓ EPI ✓
∀ user state s ∈ S, ∃ transition s → s₀ such that the exit penalty π(s → s₀) < τ_exit, where τ_exit is below the threshold of material exclusion.

Users must possess the capacity to withdraw from the system without disproportionate penalty. Irrevocable consent functions structurally as mandatory lock-in — when a system becomes a prerequisite for existence, refusal stops being feedback and becomes a survival penalty.

DPI
A non-digital path achieves equivalent outcomes (cash for payments, paper ID for identification).
EPI
A human fallback is guaranteed and accessible — automated decisions can be opted out of without consequence.
Failure
Aadhaar: opting out cuts off banking, food rations, and mobile connectivity. The penalty is not material; it is existential.
CODE Inspectability of Logic DPI ✓ EPI ✓
For any decision function f: X → Y determining outcomes, ∃ a publicly accessible artifact A_f that fully specifies f(x) for all inputs x ∈ X.

Power in a digital system resides in execution. Lessig (2006): code "determines what people can or cannot do in the first place" — regulation without appeal. If the execution path is hidden, that power is not merely unaccountable; it is structurally unobservable to those it governs.

DPI
Source code, schemas, and rule definitions are publicly inspectable — not "open standards" branding but actual executable artifacts.
EPI
LWD-R disclosure: Logic (architecture), Weights (parameters), Data (training provenance), Representation (the categories the model uses to see the world).
Failure
"Open weights" with proprietary training data; closed inference paths in welfare-eligibility models; ontological capture where the categories themselves are non-contestable.
AUDIT Independent Verification DPI ✓ EPI ✓
∀ external party P, Access(P, ε_S) = true without Authorize(Operator, P). The error rate ε_S is measurable without operator permission.

Inspection enables understanding; verification enables truth. AUDIT is the sensor in the cybernetic loop. A system is verifiable only if external, permissionless actors can test its behavior, measure error rates, and document failure modes in production — without asking the operator's permission first.

DPI
Production endpoints can be probed; logs are accessible; error rates are publishable by third parties.
EPI
Statistical bounds are continuously monitored; variety drift is measured against the deployment distribution; behavioral red-teaming is permitted.
Failure
Audit theatre — operator-controlled certifications that produce a stamp without permitting independent measurement.
GOVERN Constitutive Constraint DPI ✓ EPI ✓
∃ governance function G: Rules → Rules' such that (1) G is accessible to affected parties; (2) G is binding — operator cannot override; (3) cryptographic or legal chain of custody links system behavior to G.

Verification identifies error; Constitutive Constraint enforces correction. This is structural governance, not administrative ritual: consultations, multi-stakeholder forums, and feedback portals do not satisfy GOVERN unless their output binds the operator.

DPI
A binding RFC process with chain of custody — IETF, W3C, Linux kernel maintainership are reference patterns.
EPI
The Action Boundary Protocol: a deterministic envelope around stochastic inference, with human-in-loop required above defined thresholds.
Failure
GDoS (Governance Denial of Service): agentic systems acting faster than governance can review, overwhelming the actuator.
FORK Independent Reproduction DPI ✓ EPI ✓
An independent party can instantiate S' such that (1) no legal prohibition prevents deployment; (2) reproduction artifacts are public; (3) user state is portable: U_S → U_S'. Natural barriers do not disqualify; only constructed barriers do.

The ultimate check on power is the ability to recreate it. FORK separates the infrastructure from its current steward. LibreOffice, MariaDB, OpenSearch, Valkey all emerged when governance variety collapsed — Oracle's Sun acquisition (2010), unilateral license changes (2021, 2024). The ecosystem restored requisite variety through forking.

DPI
Code, schemas, protocols, and user state are portable — competition is not legally prohibited.
EPI
The training pipeline is reproducible: compute, data, and pipeline access are not gated by capital asymmetry alone.
Failure
Compute Capture: "open weights" without affordable retraining means FORK is theoretical, not practical. Frontier models cost hundreds of millions to retrain.

Applied: Deterministic Infrastructure (DPI)

Paper I evaluates ledgers, registries, identity systems, and payment rails against the five tests. Three claims sharpen here, each a direct critique of dominant DPI discourse.

Utility does not equal accountability. A system can be useful at population scale and structurally captured. UPI processes hundreds of millions of transactions; that scale is evidence of adoption, not of corrigibility. Aadhaar's 1.3 billion enrolments are an indictment of EXIT, not a vindication of architecture.

The DPI Trilemma. A system cannot simultaneously optimize for state Sovereignty, population Scale, and structural Neutrality. The India Stack export agenda asserts all three; the framework demonstrates this is mathematically impossible without trade-offs that the rhetoric obscures.

The Rule of the Ledger. When essential life chances are mediated through a single registry, the registry's rules become law without legislation. FEE (Functional Exit Equivalence) names the only compensatory architectural guarantee that restores requisite variety when EXIT cannot be markets-based.

Applied: Learned Systems (EPI)

Paper II extends the invariant to Epistemic Public Infrastructure — AI and agentic systems deployed in welfare, policing, judicial, and eligibility contexts. The five tests remain. Their verification changes.

LWD-R replaces "open source AI" as the CODE bar. Logic, Weights, Data, and Representation must all be disclosed — the fourth layer is the framework's principal contribution. Without R, the categories through which a system "sees" the world are proprietary, and citizens are locked into a reality they cannot define.

Variety Drift makes EPI corrigibility a persistence condition, not a deployment certification. A model that passed all five tests at deployment may fail them two years later — not because anyone changed anything, but because the world moved and the model did not. Δ(t) = V_world(t) − V_model(θ); when Δ(t) exceeds threshold, mandatory revalidation is triggered.

Compute Capture is FORK's failure mode for learned systems. Open weights without affordable retraining produce an artifact that cannot be reproduced — replacement is theoretical, not real.

The Action Boundary separates probabilistic inference from deterministic execution. Without an enforceable validation layer, agentic systems exercise unbounded epistemic power and overwhelm governance through GDoS: decisions accumulate faster than any review apparatus can correct them.

The Protocol

Schema v3.0 turns the framework into a falsifiable protocol. Operators publish declarations; independent auditors verify them. Both artifacts are JCS-canonicalized, Ed25519+DID-signed, and served from a well-known endpoint.

01 · OPERATOR
Declare
Publish infrastructure.json at /.well-known/corrigibility/ documenting the five tests for your system.
02 · AUDITOR
Verify
Fetch the declaration. Independently assess each test. Publish audit.json with signed verdicts.
03 · EVALUATOR
Compare
LLM-based assessment skills consume both manifests, surface contradictions, and rate corrigibility status.

Glossary

Ten terms readers will encounter outside the page. Full glossary in Paper I, Appendix.

Corrigibility
The structural capacity of those affected by a system to detect error, signal harm, and trigger correction. Not a moral preference but a stability requirement — without it, errors accumulate without bound.
Requisite Variety
Ashby's Law (1956): a controller must possess at least as many response states as the system it regulates. Applied to public infrastructure: governance mechanisms must match the variety of harms a system can produce.
Reversibility of Participation
Formal name for EXIT. Users must be able to withdraw without disproportionate penalty — the exit penalty must fall below the threshold of material exclusion from essential services.
LWD-R
Four layers of AI transparency required to satisfy CODE for learned systems: Logic (architecture), Weights (parameters), Data (provenance), Representation (categories the model uses to perceive the world). Omitting any one prevents external reconstruction of epistemic behavior.
Action Boundary
A deterministic envelope around stochastic inference. Validation layers separate "what the model predicts" from "what the system is permitted to do." Without it, agentic systems exercise unbounded epistemic power.
Variety Drift
Δ(t) = V_world(t) − V_model(θ): the gap between environmental variety and the variety the model can handle. Drift widens over time, even with a frozen model. AI corrigibility is therefore a persistence condition, not a deployment certification.
Compute Capture
FORK's failure mode for learned systems: when retraining costs exceed the resources of any party except the original operator, "open weights" fails practical reproduction. Capital asymmetry becomes a constructed barrier.
GDoS
Governance Denial of Service. When agentic systems act faster than any review apparatus can correct, the actuator (GOVERN) is overwhelmed by volume — the loop opens not by absence of governance but by saturation of it.
FEE
Functional Exit Equivalence. A compensatory architectural guarantee that recreates error-signal strength when EXIT is suppressed by essentiality — for example, a paper alternative legally guaranteed to deliver equivalent outcomes to the digital path.
Open-washing
Claiming openness through partial compliance — open APIs over closed governance, open weights over closed data, open standards over closed execution — while retaining structural control. Identified in Paper I as a sensor attack on accountability infrastructure.
